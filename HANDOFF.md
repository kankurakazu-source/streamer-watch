# HANDOFF.md — Claude Codeへの引き継ぎ資料

このファイルは、Claude（チャット）とのディスカッションで決まった設計意図・制約を
Claude Code（実装担当）に引き継ぐためのブリーフィングです。
ブラッシュアップ作業を始める前に、このファイル全体を読み込んでください。

---

## 1. プロジェクトの目的

X(旧Twitter)で「日本の人気ストリーマー界隈」を専門に扱う考察系アカウントを育て、
将来的に有料note・アフィリエイトへ誘導するマネタイズ導線を作ること。
今回のスコープは **「投稿ネタの自動収集・下書き生成」の部分**。

- 収益目標: 月数万円（現実的な範囲。バズ狙いの一発ではなく継続運用を重視）
- Apex Legendsなど特定ゲームには絞らず、**ストリーマー本人の動向（配信・数字・界隈の動き）**にフォーカス
- 完全自動投稿はしない。AIは下書き生成まで、投稿は必ず人間が最終判断する

## 2. アカウントのキャラクター設定

- **キャラ性のある発信**（数値だけでなく自分の考察・意見も入れる）
- 単なるデータ報告ではなく「なぜそうなったか」の解釈を加えるのが差別化ポイント

## 3. 対象配信者（初期リスト）

加藤純一、布団ちゃん、もこう、はんじょう、たいじ、バトラ、GON、Laz、柊つるぎ、関優太(SPYGEA)

- 加藤純一・布団ちゃん・もこうは「雑談・身内ノリ系」
- はんじょう・たいじ・バトラ・GON・Laz・関優太はApex Legends配信者コミュニティ寄り
  （※ただし今回はApexというゲーム軸の投稿はせず、あくまで配信者本人の動向を追う）
- 柊つるぎはVTuber/女性配信者枠

**重要**: `config.py`内のTwitchログイン名・YouTubeチャンネルIDは一部仮の値。
実装ブラッシュアップの最初のタスクとして、正確な値に置き換える必要がある。

## 4. 絶対に守るべき制約（安全面）

これらはユーザーとの合意事項であり、機能追加・改善時も必ず維持すること。

1. **未確認の噂・炎上情報を断定的に書かない**（「〜という声もある」程度のヘッジ必須）
2. **配信者本人の発言を捏造・誇張して引用しない**
3. **データは公開情報（視聴者数・配信タイトルなど）のみを使用**
4. **完全自動投稿は実装しない**（下書き生成→人間レビュー→手動投稿、の流れを崩さない）
5. X/Twitch/YouTubeそれぞれの利用規約・APIガイドラインを守る

`analyzer.py`のSYSTEM_PROMPTにこれらの制約を明文化済み。
プロンプトを変更する場合もこの制約は必ず残すこと。

## 5. 現在の実装状況

```
streamer_watch/
├── config.py                    # 対象配信者・APIキー管理（Twitchログイン名など要確認）
├── collectors/
│   ├── twitch_collector.py      # Twitch Helix APIで視聴者数取得（未実機テスト）
│   ├── youtube_collector.py     # YouTube Data APIでライブ同接取得（未実機テスト）
│   └── x_mentions_collector.py  # X API v2言及数取得（有料プラン前提、未契約なら自動スキップ）
├── storage.py                    # SQLiteで履歴保存、前回比の急増率を計算
├── analyzer.py                   # Claude APIで「事実＋考察」形式の下書きをJSON生成
├── main.py                       # 収集→保存→急増検知→下書き生成→ファイル出力を統括
├── requirements.txt
├── .env.example
└── README.md                     # セットアップ手順・運用フロー・cron設定例
```

**注意**: このコードは開発環境にネットワーク接続が無い状態で書かれたため、
構文チェックのみ実施済みで、実際のAPI呼び出しは未検証。
Claude Codeでの最初の検証ステップとして、実際のAPIキーを使った動作確認を推奨。
